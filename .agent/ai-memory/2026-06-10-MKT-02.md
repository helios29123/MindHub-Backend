# MKT-02: Tạo thông báo cho học viên trong khóa

## 1. Tóm tắt nhiệm vụ
- Xây dựng API `POST /api/instructor/course-announcements` để phục vụ việc tạo thông báo cho học viên trong khóa học của giảng viên (`Instructor`).
- Nghiệp vụ: Do cơ sở dữ liệu hiện tại (ERD GD1) chưa có bảng/cột để lưu trữ thông tin thông báo, chúng ta thống nhất thực hiện theo phương án mock: kiểm tra tính hợp lệ của dữ liệu đầu vào và kiểm tra quyền sở hữu khóa học của giảng viên, sau đó trả về HTTP status `501 Not Implemented` cùng dữ liệu mock theo đúng đặc tả kỹ thuật.

## 2. Quyết định kỹ thuật & Thiết kế
- **Xác thực & Phân quyền (Middleware)**:
  - Sử dụng middleware `['auth.session', 'role:instructor']` và prefix `instructor` để bảo vệ route.
- **Validation**:
  - Tạo mới request validator `CourseAnnouncementRequest` để xác thực:
    - `course_id` (required, exists:courses,id)
    - `title` (required, string, max:255)
    - `content` (required, string)
- **Kiểm tra quyền sở hữu khóa học**:
  - Trong `MarketingController`, tìm kiếm khóa học theo `course_id`. Xác thực xem `instructor_id` của khóa học có trùng với ID của giảng viên đang đăng nhập hay không. Nếu không trùng khớp, trả về HTTP status `403` kèm thông báo `"Bạn không có quyền thực hiện thao tác này."`.
- **Response tạm (HTTP 501)**:
  - Nếu xác thực thành công, API trả về mã lỗi HTTP `501` và chuỗi JSON mock:
    ```json
    {
      "success": true,
      "message": "Thao tác thành công",
      "data": "{\"banner_id\": 1, \"status\": \"active\"}"
    }
    ```
    Chuỗi dữ liệu mock được định dạng thông qua `CourseAnnouncementResource::resolve()`.
